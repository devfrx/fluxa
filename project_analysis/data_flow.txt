FLUSSI LOGICI:
User Input → UI → Agent → LLM Client → LMStudio API
                   ↓
              Tool Registry ← Tools Execution
                   ↓
              Memory Manager → Database
                   ↓
              Response → UI → User

ESEMPIO FLUSSO DATI:
Scenario: User chiede "Leggi il file config.py"

1. CONVERSATIONS
   └─ Nuova conversazione creata
      id=1, title="Aiuto con file"

2. MESSAGES
   ├─ Messaggio utente
   │  id=1, conversation_id=1, role="user"
   │  content="Leggi il file config.py"
   │
   └─ Messaggio assistant (con tool call)
      id=2, conversation_id=1, role="assistant"
      content="Sto leggendo il file..."
      metadata='{"tool_call_id": "tc_123"}'

3. TOOL_EXECUTIONS
   └─ Esecuzione tool file_read
      id=1, message_id=2, tool_name="file_read"
      parameters='{"path": "config.py"}'
      result="File content: ..."
      status="success"

4. MESSAGES
   └─ Messaggio finale assistant
      id=3, conversation_id=1, role="assistant"
      content="Ho letto il file. Contiene..."
